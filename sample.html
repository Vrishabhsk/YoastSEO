WP_Query

On the front end, we have pages like the home page, blog page, archive pages, and single post pages. Archive pages can include for dates, authors, tags, categories, and even custom taxonomies or custom post types.

Based on this URL structure or the pages, WP Query is used to fetch the data from the database. In this video, we will learn how the WordPress Query identifies what type of data needs to be fetched from the database based on the URL and the different types of pages. In general, WP Query can be used to query all posts or custom post types. You can also perform a tax query, a meta query, or pass many other parameters.

All of those parameters, like the author parameter, can be used to fetch posts or specific post types based on the author or author name. There are also multiple other parameters, such as category and tag parameters. You can see the full list of parameters here on the left-hand side. All these parameters can be explored in the documentation.

On developer.wordpress.org, under the WP Query class references, you can learn about all the available parameters to query the posts. You can go through it, but I won’t go over every feature here because you can easily learn about it from this documentation.

Difference between is_single & is_singular

is_single will work for all single posts and all custom post types. But, if you want to check whether the current page is a specific custom post-type page or not, you can useis_singular and pass the post type as an argument.

  • If I useis_singular('movie'), then it won’t print this WP Query object because the current page is a single post page, but it's not a post of themovie post type. There is a difference betweenis_single andis_singular.

If you just want to check whether the current page is a single post page (regardless of custom post types), you would useis_single. But if you want to check whether the single post page is about themovie post type or any other custom post type, then you can useis_singular.
